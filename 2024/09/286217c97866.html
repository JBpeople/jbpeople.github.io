1.1. try 语句常用知识

  • 精确的错误捕获放在前面；
  • try…except…else 表示代码没有错误的时候执行 else，与 finally 不同的是即使代码没有出错，但是中间 break 或者 return 了之后就不会再执行 else 了。
  • 使用空的 raise 会把当前错误原封不动的传出去；

2. 案例故事

  • 不要使用 try…except Exception… 包含一堆代码，这样失去了错误捕获的意义；
  • 自己编写相关错误对象，进行错误转换；
  • 不要抛出抽象级别高于当前模块的错误，包装抽象级别低于当前模块的异常；

3. 编程建议

3.1. 不要随意忽略异常

少用下面的代码，抛出异常都是有原因的，不做任何处理违背了抛出异常的意义。

3.2. 不要手动校验输入数据的正确性

有时候用户会输入一些东西，判断这些东西是否合理往往需要编写大量的校验代码。而如果把这些交给第三方库或者自己编写专门的校验包，将校验逻辑和主逻辑分离开简化代码很有用。

3.3. 抛出可区分的异常

  • 自定义异常名称建议都使用 Exception 或者 Error 结尾；
  • 建议都继承自 Exception，更精确的异常子类可以通过继承父类；
  • 不要使用 assert 来校验参数是否合法，这东西主要用于测试；
  • 抛出错误捕获错误都是增加开发者的心智负担，最好的设计应该是不做任何错误处理。例如空对象设计模式，该设计模式提出在不影响调用的情况下遇到错误返回类似于正确对象的特殊对象；
