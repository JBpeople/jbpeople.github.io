1.2. 列表的可变性

  • 可变：列表，字典，集合
  • 不可变：整数，浮点数，字符串，字节串，元组

对于可变变量，传入函数如果函数内部对变量值发生修改，那外部变量跟随修改。对于不可变变量，传入函数如果函数内部对变量值发生修改，外部变量不会发生任何修改。这涉及到 C 函数的两种传递模式。

  1. 值传递：调用函数的时候传递过去的是变量的克隆，所以对变量修改无法影响外部变量；
  2. 引用传递：调用函数的时候传递过去的是变量的引用（内存地址），因此对变量修改会影响外部变量；

Python 传递的是变量所指对象的引用，相当于每次调用函数前，python 新建了一个内部变量，让其和传入的变量使用 = 进行一次赋值。是否会影响外部变量只跟变量是否可变有关。

1.5. 认识字典的有序性和无序性

在 python 3.6 之前字典的顺序根据键的哈希值排序，因为其底层就是哈希表。在 3.6 之后字典出现了一个新的特性，字典中的顺序跟添加进字典的先后有关系，字典变成有序的了。但是字典的顺序并不会成为判断字典是否相等的必要条件。

1.7. 对象的可哈希性

  1. 所有的不可变内置类型，都是可哈希的，比如 str、int、 tuple、frozenset 等；
  2. 所有的可变内置类型，都是不可哈希的，比如 dict、list 等；
  3. 对于不可变容器类型 (tuple, frozenset)，仅当它的所有成员都不可变时，它自身才是可哈希的；
  4. 用户定义的类型默认都是可哈希的；

3. 编程建议

3.1. 按需返回而不是全部一次性生成

在 python 2 中使用 range 一个非常大的数字会非常慢，因为 python 2 返回了一整个列表。而在 python 3 中则会非常快，因为该方法被改成了返回一个惰性计算对象。只有迭代的时候才会按需生成数字。

3.2.1. 避开列表的性能陷阱

python 的列表类型对象，其底层使用的是 array 数组，该元素的特点就是如果是一长串数字，从第一位插入数字会让后面每个数字往后移动一位，效率及其低下。

如果有从第一位插入数字的需求应该换作 collections 里面的 deque，该底层使用了双端队列，从头插入和从尾插入效率一样快。

还有一种性能陷阱是判断元素是否在列表内，因为列表是一种数组结构，判断元素在列表内需要从左到右依次对比。如何查询元素是否在容器内参考下面方式。

3.4. 使用有序字典去重

集合去重容易导致原有的顺序错乱，这是哈希结构决定的。使用有序字典去重，则可以保留结构的原有顺序。

3.5. 不要一边修改列表一边遍历列表

因为删除了列表元素导致列表的长度在缩减，但是循环的索引次数在增加导致有些元素压根没有被遍历到。如果使用一个新的列表来存储就不会出现这些问题。
