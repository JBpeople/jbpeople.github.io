3. 编程建议

3.1. 不必预计算字面量表达式

这样“胡乱”数字虽然说加上注释可以看懂，但是没有写成这样 11 * 24 * 3600 清晰。这样写并不会影响 Python 的运行效能。Python 虽说是解释性语言但是在运行前仍然会有一个加速编译的过程，将代码编译成二进制字节，使用 dis 模块可以把二进制反汇编成人类可理解的内容。

可以看到 LOAD_CONST 实际上已经自动将字面量计算出来了，所以无论调用该方法多少次，只能会在编译时间计算一次。

3.5. 字符串拼接效率

在 Python 中因为字符串是可变变量，所以如果使用 += 每次都会重新生成新对象，重新分配内存。但是在 Python 2.2 之后因为该拼接方法经常使用所以大幅优化了执行效率。
