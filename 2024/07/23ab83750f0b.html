1.1. 变量常见用法

单下划线命名。在常用的诸多变量名中，单下划线 _ 是比较特殊的一个。它常作为一个无意义的占位符出现在赋值语句中。_ 这个名字本身没什么特别之处，这算是大家约定俗成的一种用法。

1.2. 给变量注明类型

Python 是动态类型语言，使用变量时不需要做任何类型声明，但是代码缺乏阅读性。在 Python 3.5 之后加入了类型注解，用以标明变量类型。

[!warning]
类型注解仅仅用以表明变量类型，不会对变量做校验。如果需要使用静态校验，需要配合 mypy 模块使用。

1.3. 变量命名规则

  • 对于普通变量，使用蛇形命名法，比如 max_value;
  • 对于常量，采用全大写字母，使用下划线连接，比如 MAX_VALUE;
  • 如果变量标记为“仅内部使用”，为其增加下划线前缀，比如 _local_var;
  • 当名字与 Python 关键字冲突时，在变量末尾追加下划线，比如 class_。

[!note]
不要一昧为了追求变量的描述性而忽视变量长度。

变量匹配类型。

变量类型 命名方式
布尔类型变量 用 is，has，allow 开头，例如 has_error。
数字类型变量 使用中文注释为数字的单词，例如 port，age，radius，使用 _id 结尾，例如 user_id，使用 length，count 开头结尾，例如 length_of_username。

[!note]
数字类型变量尽量不要使用复数，例如 apples，容易和列表字典类型重复。

1.4. 注释

注释作为代码之外的说明性文字，应该尽量提供那些读者无法从代码里读出来的信息。描述代码要这么做，而不是简单复述代码本身。

[!warning]
不要使用注释来屏蔽代码，容易导致代码不易阅读。不要的代码因立即删除，使用 git 来做历史版本管理。

2. 编程建议

2.1. 保持变量的一致性

定义变量需要保持名字一致性与类型一致性。

名字一致性，在同一个项目的不同模块下不应该变来变去，如果你把项目里的 用户头像 叫作 user_avatar_url，那么在其他地方就别把它改成 user_profile_url。否则会让读代码的人犯迷糊: user_avatar_url 和 user_profile_url 到底是不是一个东西。

类型一致性，不要把同一个变量重复指向不同的值。

2.2. 变量定义应该尽可能靠近使用

不要提前把所有变量定义了然后再使用，应该一边使用一边定义。

2.6. 不要使用locals

在python 中 local 关键字作用用于暴露作用域内的所有变量。

使用 locals 作为返回值可以少写一些代码，但是对于阅读者来说需要记住作用域内的所有变量，这几乎是不可能的事情。

2.7. 空行也是注释

在写代码时，我们可以适当地在代码中插入空行，把代码按不同的逻辑块分隔开，这样能有效提升代码的可读性。

2.8. 先写注释，后写代码
